Analytics Queries

QUERY 01
Rolling 5-Match Form Table
"How has United's form evolved across the season?"
Window Function STRING_AGG CTE
▾
-- Rolling 5-match form: W/D/L string + points for last 5 games
WITH match_results AS (
  SELECT
    m.match_date, m.stage,
    CASE WHEN m.home_team_id = 1 THEN m.home_goals ELSE m.away_goals END AS utd_goals,
    CASE WHEN m.home_team_id = 1 THEN m.away_goals ELSE m.home_goals END AS opp_goals
  FROM matches m
  WHERE m.home_team_id = 1 OR m.away_team_id = 1
),
rolling_form AS (
  SELECT *,
    STRING_AGG(
      CASE
        WHEN utd_goals > opp_goals THEN 'W'
        WHEN utd_goals = opp_goals THEN 'D'
        ELSE 'L'
      END, '' ORDER BY match_date
    ) OVER (
      ORDER BY match_date
      ROWS BETWEEN 4 PRECEDING AND CURRENT ROW
    ) AS last_5_form
  FROM match_results
)
SELECT match_date, stage, last_5_form
FROM rolling_form ORDER BY match_date;
QUERY 02
Bruno Fernandes by Formation
"Is Bruno more effective as a #10 in 4-2-3-1 than in a 4-3-3?"
JOIN GROUP BY ROUND / NULLIF
▾
SELECT
  tf.formation_name,
  COUNT(DISTINCT pms.match_id)           AS matches,
  SUM(pms.goals + pms.assists)            AS goal_contributions,
  ROUND(AVG(pms.key_passes), 1)          AS avg_key_passes,
  ROUND(
    SUM(pms.passes_completed)::NUMERIC
    / NULLIF(SUM(pms.passes_attempted), 0) * 100, 1
  )                                         AS pass_accuracy_pct,
  ROUND(AVG(pms.rating), 2)               AS avg_rating
FROM player_match_stats pms
JOIN tactical_formations tf
  ON tf.match_id = pms.match_id
  AND tf.team_id = pms.team_id
  AND tf.is_starting = TRUE
WHERE pms.player_id = 8  -- Bruno Fernandes
GROUP BY tf.formation_name
ORDER BY avg_rating DESC;
QUERY 03
Home vs Away Performance Split
"Is Old Trafford still a fortress, or are United just as leaky at home?"
CASE WHEN GROUP BY Points Per Game
▾
SELECT
  venue_type,
  COUNT(*)                                                  AS matches,
  SUM(CASE WHEN utd_goals > opp_goals THEN 1 ELSE 0 END)  AS wins,
  SUM(utd_goals) - SUM(opp_goals)                        AS goal_difference,
  ROUND(AVG(utd_xg), 2)                                   AS avg_xg_for,
  ROUND(AVG(opp_xg), 2)                                   AS avg_xg_against,
  ROUND(
    SUM(CASE WHEN utd_goals > opp_goals THEN 3
             WHEN utd_goals = opp_goals THEN 1
             ELSE 0 END)::NUMERIC / COUNT(*), 2
  )                                                         AS points_per_game
FROM utd_matches
GROUP BY venue_type;
QUERY 04
xG Overperformance Analysis
"Which matches did luck decide? Where is United punching above its weight?"
xG Model CASE WHEN Derived Columns
▾
SELECT
  match_date, opponent, actual_goals || '-' || opp_goals AS score,
  xg AS utd_xg,
  ROUND(actual_goals - xg, 2)   AS xg_overperformance,
  CASE
    WHEN actual_goals - xg >  0.5 THEN 'Outperformed xG'
    WHEN actual_goals - xg < -0.5 THEN 'Underperformed xG'
    ELSE 'In line with xG'
  END                             AS xg_verdict,
  CASE
    WHEN ROUND(xg) > ROUND(opp_xg) THEN 'xG Win'
    WHEN ROUND(xg) = ROUND(opp_xg) THEN 'xG Draw'
    ELSE 'xG Loss'
  END                             AS xg_expected_result
FROM xg_analysis
ORDER BY xg_overperformance DESC;
QUERY 05
Points Dropped from Winning Positions
"How often did United lead and fail to hold on?"
Window Function SUM OVER LEFT JOIN
▾
WITH running_score AS (
  SELECT
    me.match_id, me.minute,
    SUM(CASE WHEN me.team_id = 1 AND me.event_type = 'goal'
             THEN 1 ELSE 0 END)
      OVER (PARTITION BY me.match_id ORDER BY me.minute) AS utd_running,
    SUM(CASE WHEN me.team_id != 1 AND me.event_type = 'goal'
             THEN 1 ELSE 0 END)
      OVER (PARTITION BY me.match_id ORDER BY me.minute) AS opp_running
  FROM match_events me
),
had_lead AS (
  SELECT match_id,
    MAX(CASE WHEN utd_running > opp_running THEN 1 ELSE 0 END) AS was_winning
  FROM running_score
  GROUP BY match_id
)
SELECT
  fr.opponent, fr.final_score,
  CASE
    WHEN hl.was_winning = 1 AND result = 'L' THEN 'Points dropped (loss)'
    WHEN hl.was_winning = 1 AND result = 'D' THEN 'Points dropped (draw)'
    WHEN hl.was_winning = 1               THEN 'Lead held'
    ELSE 'Never led'
  END AS outcome
FROM final_results fr
LEFT JOIN had_lead hl ON hl.match_id = fr.match_id;
QUERY 10
Recursive Season Standings Builder
"Build the cumulative league table matchweek-by-matchweek."
Recursive CTE RANK() LAG()
▾
WITH RECURSIVE matchweeks AS (
  SELECT 1 AS mw
  UNION ALL
  SELECT mw + 1 FROM matchweeks WHERE mw < 38
),
cumulative AS (
  SELECT
    s.team_id, t.name AS team_name, s.matchweek,
    s.points, s.goal_diff, s.goals_for,
    RANK() OVER (
      PARTITION BY s.matchweek
      ORDER BY s.points DESC, s.goal_diff DESC, s.goals_for DESC
    ) AS position
  FROM standings s
  JOIN teams t ON t.team_id = s.team_id
)
SELECT
  matchweek, position, team_name, points,
  LAG(position) OVER (
    PARTITION BY team_id ORDER BY matchweek
  )                               AS prev_position,
  position - LAG(position) OVER (
    PARTITION BY team_id ORDER BY matchweek
  )                               AS position_change
FROM cumulative
ORDER BY matchweek, position;

About this Project

This project demonstrates real-world SQL skills applied to a domain that matters — professional football analytics. Every design decision mirrors what actual sports data companies like Opta and StatsBomb use in production.

The schema is fully normalized to 3NF, uses ENUMs for controlled vocabularies, and includes generated columns (goal_diff, points) to guarantee data integrity without application-layer computation.

Queries go beyond basic SELECTs. They answer the questions a real United analyst or data journalist would ask — from Bruno's formation dependency to the statistical cost of Casemiro's red cards.

Materialized views are pre-aggregated for dashboard performance, and all performance-critical columns are indexed. The project is ready to connect to Metabase, Grafana, or any BI tool.
